TWINS ERP — Updated Roadmap

PHASE 0 — Locked Architectural Decisions

These are already decided:

Inventory costing
	•	Company supports:
	•	weighted_average
	•	specific_lot
	•	weighted_average scope = depot + product
	•	specific_lot = batch/lot-based

Storage design
	•	companies.costing_method = current preference
	•	inventory_periods.costing_method = historical truth
	•	inventory_movements.period_id required for posted movements
	•	inventory_consumptions.period_id required for consumptions

Period policy
	•	Only one OPEN inventory period per company
	•	Posting always goes into the current OPEN period
	•	Closed periods are immutable
	•	No backdating into closed periods
	•	Corrections happen via current-period adjustments

Costing change policy
	•	If company has no inventory movements → costing method can be changed directly
	•	If company has any inventory movements → costing method locked
	•	Change then happens only via New Inventory Period Wizard

Purchase policy
	•	Purchases are postponed until this foundation is built
	•	We already locked the future purchase workflow, but we are not building it yet

PHASE 2 — Core Foundation Services

This is the policy layer.

2.1 PeriodResolver

Responsibilities:
	•	fetch current OPEN period
	•	if none exists and safe:
	•	auto-create Period 1 from companies.costing_method

2.2 PostingGate

Responsibilities:
	•	block posting if company is paused
	•	block posting if no valid open period exists
	•	ensure posting never lands in a closed period

2.3 InventoryPostingContext

Standard context for posting:
	•	company_id
	•	period_id
	•	costing_method
	•	user_id
	•	posting_key
	•	posting_group_uuid

PHASE 6 — New Inventory Period Wizard

This is the controlled cutover.

6.1 Wizard flow
	1.	Acquire company lock
	2.	Pause posting
	3.	Validate stable state
	4.	Choose new costing method
	5.	Choose effective timestamp
	6.	Close current period
	7.	Snapshot inventory
	8.	Create new open period
	9.	Post opening balances
	10.	Resume posting
	11.	Show audit result

6.2 Snapshot source

Use:
	•	depot_stocks as authoritative current stock snapshot

6.3 Opening balance rules

Opening balance movements:
	•	belong to new period
	•	recreate same physical stock
	•	preserve value continuity

6.4 WA → Specific Lot

If stock is commingled:
	•	create synthetic Opening Lot batches
